Emacs Step by Step

零、废话

在下使用emacs也有些时日了，从最初的学习各种key bindings到现在自己写一些短的elisp代码来定制自己的emacs，还是经历了一个比较曲折的过程的。期间也尝试过使用vi，感觉vi也是个很高效很优秀的编辑器，但是最后还是沉迷于eamcs了。原因有三。首先，我希望随着使用的深入，编辑器和自己的契合程度越来越高。其次，我自己本身就很喜欢lisp。最后，emacs多多少少可以用来装逼。:)

开个玩笑。emacs是一个优秀的开发环境，但是并不容易驾驭。虽然可以通过elisp来将其定制到比eclisp/vs还智能，但是确实需要一个很长的学习曲线，而vi则是反其道而行之——在最短的时间内最高效地完成工作。因此，若要用好emacs，一颗折腾的心是必须的。而折腾可以让你进步——还能让你成为一个hacker。

书归正传。写此文的导火索是之前在知乎上回答了一个问题《说说每个人使用emacs的境界》。境界这个东西很难讲，但是学习还是有一个过程的。特别是很多新人在学习一个新东西的时候，更是需要一个roadmap来指引方向，所以我整理了一个list，并且将其作为提纲收集了一些学习资料。当然，鉴于我自己的emacs水平还处于一个不上不下的程度，目前我还不能提供更多的帮助。但是随着深入学习，我也会持续更新这篇文章。

一、入门

首先，你需要阅读一些入门教程。新手应该跟随一两个教程并且尝试使用emacs。一定要实际操作，学习编辑器的使用更多是通过练习而不是记忆。

  • Emacs Beginner's HOWTO
  • A Tutorial Introduction to GNU Emacs
  • Living in Emacs
  • Emacs Editing Environment

二、理解概念

学会了基本使用之后，你应该已经可以使用emacs来编辑文本文件或者代码了。现在是时候尝试理解emacs里的一些关键概念了，你可以在emacs官方手册里找到这些概念及其解释。

  • GNU Emacs Manual

三、高级操作

现在，你可以尝试一些高级的操作了。有些可能比较奇怪，但是熟练使用后，你会发现这些功能比较奇怪。掌握了这些“奇因巧技”之后，你的编辑效率会很高。

四、扩展插件

这是emacs最强大的部分。vi和emacs这种编辑器之所以是专门为程序员设计的，是因为他们允许程序员编写插件进行扩展。相比vi，emacs扩展自身的能力更为强大，这得益于其内建的一门完整的编程语言——elisp。elisp是一种lisp方言，而lisp本身就是极为强大的。除此之外，emacs社区十分活跃，有成千上万已经写好的插件供你使用。只需要稍稍配置，你就能获益良多。

  • Emacs Wiki
  • Planet Emacsen
  • Emacs中文网

五、Emacs Lisp入门

现在是开始尝试自己定制并且扩展你的emacs的时候了。作为一种lisp方言，elisp可以打开你的编程视野。对lisp的褒奖太多，我就不赘述了。

  • An Introduction to Programming in Emacs Lisp
  • Xah Emacs Lisp Tutorial

六、更多

于是，现在你可以算是一个experienced emacser了。但是，你应该更加深入学习emacs，这会继续提高你的工作效率，直到达到一种武侠小说里的大侠们“人剑合一”的境界。有机会的话，我也会就某个话题写一点深入研究emacs的文章，也希望你可以跟我分享你的观点。
